DeidentifyConfig getReidentifyConfig()

Configuration for the re-identification of the content item.

This field shares the same proto message type that is used for de-identification, however its usage here is for the reversal of the previous de-identification. Re-identification is performed by examining the transformations used to de-identify the items and executing the reverse. This requires that only reversible transformations be provided here. The reversible transformations are: - CryptoReplaceFfxFpeConfig
